# Purpose

`FUNC00757` 用于查看 Prober 设备、绑定 Tester、当前产品、Tester 实际板卡、产品工艺所需板卡，以及实际比所需多出的板卡。

# Runtime Path

```text
FrmGeneralReport
  -> FuncID = FUNC00757 时走专用后端路由
  -> Func00757ReportManagement#select
  -> Func00757ServiceExt.selectFunc00757Report(param)
  -> QueryService.getDataByCustomQueryID("FUNC00757_SELECT", "0001", param)
  -> Func00757CardGapCalculator.calculateCardGap(TESTER_CARD_DETAIL, REQUIRED_PC_CARD)
```

# Inputs

查询参数来自通用报表条件区，关键过滤包括：

| Parameter | Meaning |
|---|---|
| `EQP_GROUP_ID` | Prober 设备组。 |
| `EQP_ID` | Prober 设备编号。 |
| `MODEL_ID` | Prober 型号。 |
| `TESTER_MODEL_ID` | Tester 型号。 |
| `CARD_TYPE` | 板卡类型过滤，按逗号拆分后匹配 `TESTER_CARD_DETAIL`。 |

# Outputs

| Column | Source / Meaning |
|---|---|
| `EQP_ID` | Prober 设备。 |
| `MODEL_ID` | Prober 型号。 |
| `PROD_ID` | 当前产品，来自 [EQP_STATE](../tables/eqp-state.md)。 |
| `TESTER_EQP_ID` | 绑定 Tester，来自 [ET_PROBER_TESTER](../tables/et-prober-tester.md)。 |
| `TESTER_CARD_DETAIL` | Tester 实际板卡展示，来自 [CUST_CARD_INFO](../tables/cust-card-info.md)。 |
| `REQUIRED_PC_CARD` | 产品工艺所需板卡，来自 [ET_PROC_CARD](../tables/et-proc-card.md)。 |
| `CARD_GAP` | Java 计算的实际多出板卡，见 [CARD_GAP](../metrics/card-gap.md)。 |

# Notes

`TESTER_CARD_DETAIL` 与 `REQUIRED_PC_CARD` 既是前端展示列，也是 Java 计算输入。运行时 `CUST_QUERY.FUNC00757_SELECT[0001]` 必须保留这两个列。
